# 🔒 Security Policy

## Supported Versions

We actively maintain and provide security updates for the following versions:

| Version         | Supported          |
|-----------------|--------------------|
| 0.10.x (latest) | ✅ Yes             |
| < 0.10.0        | ❌ No              |

If you are running an older version, please upgrade to the latest release to ensure you receive security patches and fixes.

---

## Reporting a Vulnerability

We take security issues seriously. If you discover a vulnerability in **FluentSignalCopier**, please follow these steps:

1. **Do not open a public GitHub issue.**
   - This helps protect users while the issue is being investigated.

2. **Report privately via email:**
   - 📧 [o.joisten@live.se](mailto:o.joisten@live.se)
   - Alternatively, use GitHub’s [Private Vulnerability Reporting](https://docs.github.com/en/code-security/security-advisories/guidance-on-reporting-and-writing/privately-reporting-a-security-vulnerability)

3. **Provide details in your report:**
   - Steps to reproduce the issue
   - Potential impact
   - Any suggested mitigations

We will acknowledge receipt of your report within **48 hours**, and provide regular updates as we investigate.

---

## Responsible Disclosure

- Please allow us time to investigate and release a patch before publicly disclosing.
- We aim to resolve confirmed security issues within **14 days**, depending on severity.

---

## Security Best Practices for Users

- Always test on a **demo account** before live trading.
- Keep your **Telegram API credentials** and **MT5 account details** private.
- Use the latest stable release for maximum protection.
- Avoid modifying the EA or bridge code unless you understand the risk implications.
